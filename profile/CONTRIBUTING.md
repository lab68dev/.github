## üîß Making Changes (Pull Requests)

Before you start coding, consider discussing your idea in an issue first ‚Äî it helps prevent duplicated effort.

### Steps:
1. **Fork the repository.**
2. **Create a new branch** for your change:
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**, following the existing code style and structure.
4. **Write clear commit messages:**

   ```bash
   git commit -m "Add: short, descriptive message"
   ```
5. **Push your branch** to your fork:

   ```bash
   git push origin feature/your-feature-name
   ```
6. **Open a Pull Request** against the main branch of this repository.

### Before Submitting:

* Run all existing tests (if available).
* Update documentation and examples as needed.
* Ensure your code follows the existing formatting conventions.
* Add relevant comments and type hints if applicable.
